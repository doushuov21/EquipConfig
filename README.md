# 设备匹配系统 - 数字孪生

## 项目简介

本项目是一个设备匹配工具，用于将后端返回的设备台账（result.json）与数字孪生导出的模型信息（data.json）进行匹配。支持手动匹配、自动匹配、Focus 聚焦定位，以及像素流视频集成。

## 文件说明

| 文件名 | 说明 |
|--------|------|
| `match.html` | 主页面，设备匹配界面 |
| `video-player.html` | 像素流视频播放页面 |
| `video.html` | 像素流视频示例页面 |
| `result.json` | 后端返回的设备台账数据 |
| `data.json` | 数字孪生导出的模型信息（UTF-16 编码） |
| `peer-stream.js` | 像素流媒体连接库 |
| `server.py` | 本地 HTTP 服务器 |

## 功能特性

### 1. 设备匹配
- **自动 ID 匹配**：页面加载时自动根据设备 ID 匹配已配置的设备
- **手动匹配**：点击未匹配设备，从候选列表中选择数字孪生设备
- **智能推荐**：按名称相似度排序推荐候选设备
- **取消匹配**：已匹配的设备可取消匹配

### 2. Focus 聚焦定位
- 点击候选设备名称旁的 📍 图标，发送 Focus 消息到数字孪生
- 帮助匹配人员快速定位设备位置
- 消息格式：
```json
{
    "action": "Focus",
    "des": "孪生体聚焦定位",
    "twinId": "设备 ID",
    "time": "时间戳",
    "isOpen": true,
    "ext": {"duration": 1}
}
```

### 3. 像素流视频
- 支持配置 IP 和端口连接像素流
- 使用 iframe 嵌入 video-player.html
- 实时显示 WebRTC 统计信息（QP 值、帧率、带宽等）

### 4. 数据导出
- **保存匹配**：导出匹配结果为 JSON 文件
- **导出 data.json**：用台账 ID 替换设备 GUID，保持原格式（UTF-16 编码）

## 快速开始

### 1. 启动本地服务器
```bash
python server.py
```

服务器会自动打开浏览器访问 `http://localhost:8000/match.html`

### 2. 手动访问
```
http://localhost:8000/match.html
```

### 3. 连接像素流
1. 在右侧视频面板输入像素流 IP 和端口
2. 点击"连接"按钮
3. 等待视频加载（默认地址：`ws://192.168.1.4:10090`）

## 操作指南

### 设备匹配流程

1. **查看匹配状态**
   - 绿色标记：已匹配设备
   - 红色标记：未匹配设备

2. **手动匹配设备**
   - 点击左侧未匹配设备
   - 在中间面板查看推荐候选列表
   - 点击 📍 图标聚焦定位设备
   - 选择正确的数字孪生设备

3. **自动匹配建议**
   - 点击"自动匹配"按钮
   - 查看系统推荐的匹配（相似度 ≥ 60%）
   - 点击"全部应用"批量应用匹配

4. **取消匹配**
   - 选择已匹配设备
   - 点击设备信息下方的"取消匹配"按钮

### 导出匹配结果

1. **保存匹配记录**
   - 点击"保存匹配"按钮
   - 下载 `matches_YYYY-MM-DD.json` 文件

2. **导出 data.json 格式**
   - 点击"导出 data.json"按钮
   - 下载 `data_matched_YYYY-MM-DD.json` 文件
   - 设备 GUID 已替换为台账 ID

## 技术细节

### 数据格式

**result.json** - 设备台账
```json
{
    "success": true,
    "result": [{
        "name": "设备名称",
        "id": "设备 ID",
        "type": "equip|struct",
        "parent": "父节点 ID",
        "children": [...]
    }]
}
```

**data.json** - 数字孪生模型（UTF-16 编码）
```json
[{
    "name": "设备名称",
    "设备 GUId": "GUID",
    "设备 Id": "ID",
    "位置": {"x": 0, "y": 0, "z": 0},
    "区域": "区域名称"
}]
```

### 匹配逻辑

1. **ID 匹配**：比较设备 ID 与数字孪生设备的 GUID/ID
2. **名称相似度**：
   - 完全匹配：100%
   - 关键部分匹配（去除数字和#）：90%
   - 包含关系：80%
   - 字符重叠度：按比例计算

### 像素流集成

- 使用 `peer-stream.js` 库连接 WebRTC
- 通过 iframe 嵌入独立视频页面
- URL 参数传递 WS 地址：`video-player.html?ws=ws://IP:PORT`

## 常见问题

### 1. 像素流连接失败
- 检查 IP 和端口是否正确
- 确保像素流服务已启动
- 检查浏览器是否支持 WebRTC
- 使用 Chrome 100+ 浏览器

### 2. 数据加载失败
- 确保 `result.json` 和 `data.json` 文件存在
- 检查文件编码（data.json 为 UTF-16）
- 使用 `server.py` 启动 HTTP 服务器（不能直接打开 HTML）

### 3. Focus 消息不生效
- 确保像素流已连接
- 检查数字孪生系统是否支持 Focus 消息
- 查看浏览器控制台日志
